# 웹툰 번역용 IME 사전 관리 & 번역 지식베이스 시스템

## 프로젝트 개요

한국어 → 번체중국어 웹툰 번역 작업을 위한
개인용 번역 워크플로우 관리 툴을 개발한다.

이 프로젝트의 핵심 목적은:

- Microsoft Bopomofo(新注音) 사용자 사전 관리
- 작품별 고유명사 관리
- 번역 용어 검색
- 번역 표현 아카이빙
- 번역 일관성 유지
- 입력 시 후보 선택 감소

이다.

이 프로젝트는 일반 사전 앱이나 CAT Tool이 아니라,
"웹툰 번역 생산성 최적화 도구"를 목표로 한다.

---

# 핵심 구조

시스템은 크게 2개의 모듈로 분리한다.

1. IME 사전 관리 모듈
2. 번역 지식베이스 모듈

두 모듈은 데이터 성격이 완전히 다르므로
반드시 분리된 구조로 설계한다.

---

# Module A — IME 사전 관리

## 목적

Microsoft Bopomofo 사용자 사전 TXT로 export될 데이터를 관리한다.

예시:

- 캐릭터 이름
- 지역명
- 조직명
- 무협 용어
- 작품 고유명사

---

# 필수 기능

## 1. 작품 단위 관리

작품별로 사전을 분리 관리한다.

예시:

```txt
[무림지존]
류서한
류서령
부각주
환수문

[학원액션]
김도훈
최민우
동성파
```

---

## 2. 작품 활성화 / 비활성화

현재 작업 중인 작품만 활성화 가능해야 한다.

예시:

```txt
☑ 무림지존
☐ 학원액션
```

활성화된 작품만 최종 IME TXT export에 포함된다.

---

## 3. 자동 약칭 생성

캐릭터 이름의 약칭을 자동 생성한다.

예시:

| 원본 | 자동 생성 |
|---|---|
| 柳胥寒 | 胥寒 |
| 尹濟恩 | 濟恩 |
| 白慕川 | 慕川 |

규칙:

- 기본적으로 마지막 2글자 사용
- 사용자 수동 수정 가능
- 약칭 삭제 가능

---

## 4. 중복 단어 감지

여러 작품에서 동일 단어가 존재할 경우 경고 표시.

예시:

```txt
지호
```

---

## 5. Microsoft Bopomofo TXT Export

Microsoft Bopomofo 사용자 사전 형식을 유지해야 한다.

예시:

```txt
柳胥寒  ㄌㄧㄡˇ ㄒㄩ ㄏㄢˊ
```

필수 조건:

- UTF-8 유지
- 공백 형식 유지
- BOM 깨짐 방지
- 줄바꿈 유지
- 원본 호환성 유지

---

## 6. 대량 Import

다음 형식 지원:

- TXT
- CSV
- 여러 줄 붙여넣기

---

# Module B — 번역 지식베이스

## 목적

IME에는 넣지 않지만
번역 참고용으로 사용할 표현/용어를 관리한다.

예시:

| 한국어 | 번체중국어 |
|---|---|
| 인싸 | 潮人 / 社交達人 |
| 아싸 | 邊緣人 |
| 한심 | 沒出息 |

---

# 지식베이스 카테고리

## 1. 번역 표현

예시:

```txt
취향저격 → 命中喜好
```

---

## 2. 정자/표기 관리

예시:

| 비권장 | 권장 |
|---|---|
| 기록 | 紀錄 |
| 비고 | 備註 |

---

## 3. 게임 용어

예시:

| 한국어 | 대만 용어 |
|---|---|
| 챌린저 | 菁英 |
| 마스터 | 大師 |

---

## 4. 밈 / 인터넷 용어

예시:

```txt
인싸
아싸
```

---
